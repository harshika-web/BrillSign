The Audit Evidence Lifecycle

Every signing event passes through a 5-stage evidence pipeline — from capture to independent verification.

Event Captured

Every signing action — open, view, sign, reject — is logged with full metadata and timestamps.

Cryptographic Seal

Each event is immediately hashed with SHA-256. The seal is immutable — any modification breaks the chain.

Chain Linked

Events are back-linked into a chronological chain. Each entry references the previous hash, preventing insertion or deletion.

Evidence Packaged

A complete audit evidence bundle is generated on demand — containing the full signing lifecycle with all proofs.

Independent Verification

Any third party can verify the evidence without BrillSign access — using standard cryptographic tools.

Inside an Evidence Package

Every BrillSign audit evidence bundle contains a complete, cryptographically verifiable record of the entire signing lifecycle.

  • Signer Identity Proof Hash — Zero-knowledge cryptographic proof of signer identity.
  • RFC 3161 Timestamp Chain — Legally recognised timestamps from certified authorities.
  • Document Integrity Seal — SHA-256 hash of the signed document at the moment of signing.
  • Geolocation & IP Metadata — Signer location + network data for regulatory evidence.
  • Device Attestation Record — Hardware identity + platform attestation from the signing device.
