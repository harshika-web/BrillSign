CLI Quick Start

Verify in one command — zero dependencies

Install the BrillSign CLI and run a single verification command. The tool checks document integrity, signer identity, timestamp, and ledger anchor — all locally, with no account required.

Terminal — bash
# Install the BrillSign CLI
$ npm install -g @brillsign/cli
 
# Verify a document signature
$ brillsign verify ./contract_signed.pdf
 
Checking document integrity...
✓ HASH: sha256: 7f3a9b2c...d84f1e09
✓ HASH MATCH: VERIFIED — document untampered
 
Checking signer identity...

REST API Reference

Integrate verification into any system

GET
/v3/verify/{document_id}
Verify a document by its BrillSign ID. Returns full proof status, signer identity, and timestamp.
GET
/v3/verify/hash/{sha256}
Check document integrity by submitting a hash. Works without platform access.
POST
/v3/verify/upload
Upload a document file directly. Computes hash server-side and returns the full verification result.
GET
/v3/proof/{proof_id}
Retrieve a complete non-repudiation proof package including all binding artifacts.

Open Standard Verification

Verify with no proprietary tools

OpenSSL — PKCS#7 Signature

openssl cms -verify \
-in proof.p7s \
-inform PEM \
-CAfile brillsign-ca.pem

Validate the CMS/PKCS#7 signature using the BrillSign public CA certificate. No account required.

GnuPG — Public Key Verify

gpg --verify \
proof.sig \
contract.pdf

Use GnuPG to verify the detached signature against the document and BrillSign's public key from the public keyserver.

SHA-256 Hash Check

sha256sum contract.pdf
# Compare to anchored hash
# from proof package JSON

Manually compute the document hash and compare it to the anchored proof hash. Completely offline, zero dependencies.
