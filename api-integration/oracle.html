How It Works

From Oracle Workflow to Signed & Archived.

Oracle Triggers via CLM or Fusion

A contract or PO reaches "Signature Required" in Oracle CLM or Fusion Procurement, triggering BrillSign via REST API.

2

ZK-Signed by All Parties

BrillSign routes to all required signers. Each signs without needing an Oracle account. Fully ZK-encrypted and legally binding.

Oracle Status Updated, Proof Archived

CLM/Fusion contract status advances to "Executed." Cryptographic proof archives to Oracle's document repository automatically.
