Architecture

Event-Driven Signing on AWS Infrastructure.

1

Lambda Calls BrillSign API

Your Lambda function uploads a document and creates a signing request using the BrillSign Python or Node SDK.

2

BrillSign ZK-Processes & Routes

Signers complete signing. BrillSign fires a HMAC-verified webhook to your API Gateway endpoint on completion.

3

Lambda Handles + S3 Stores

Your triggered Lambda verifies the signature, stores the document in S3, and continues your business workflow.

Setup

Deploy the Reference Architecture.

1

Install the SDK

Add BrillSign to your Lambda layer or deployment package.

pip install brillsign
2

Store API Key in Secrets Manager

Put your BrillSign API key in AWS Secrets Manager. Retrieve it at Lambda init time — never hardcode.

aws secretsmanager create-secret --name brillsign/api-key --secret-string "sk_live_..."
3

Configure Webhook → API Gateway

Register your API Gateway URL as a BrillSign webhook endpoint. BrillSign signs all payloads with HMAC-SHA256 for WAF-safe verification.

4

S3 Bucket Policy

Grant BrillSign's IAM role s3:PutObject on your target bucket, or use pre-signed URLs to push documents directly from BrillSign to S3.

Use Python's hmac module with a timing-safe comparison. BrillSign sends the signature in the X-BrillSign-Signature: sha256=... header. Our Python SDK includes a ready-made brillsign.webhooks.verify() helper that handles this correctly.
