Core Infrastructure

Six pillars of sovereign security

IPFS Node Distribution

Documents are sharded across a global network of InterPlanetary File System nodes. No single server holds your complete file — it's split across geographies automatically.

Zero-Knowledge Proofs

BrillSign verifies document authenticity without ever accessing the document's contents. Mathematical proofs confirm validity while preserving full privacy.

Auto-Heal Redundancy

If any node goes offline, the network automatically reconstructs missing shards from distributed peers before your next access request — zero downtime, zero data loss.

Dilithium-3 Encryption

Post-quantum resistant lattice-based encryption secures every document shard. Future-proofed against the threat of quantum computing decryption attacks.

Immutable Timestamps

Every signing event is anchored to a decentralized ledger with a cryptographic timestamp that cannot be retroactively altered by any party — including BrillSign itself.

Sovereign Access Control

Only cryptographic key holders can authorize document access. No administrator can override this — not BrillSign staff, not third-party auditors, not government agencies.

Three-tier encryption model

Every document passes through three independent cryptographic layers before any shard is distributed.

T1

Content Sealing — SHA-3 / 512-bit Hash

The document binary is hashed using SHA-3 (512-bit) creating an immutable fingerprint before any network transmission occurs.

At Upload
T2

Shard Encryption — AES-256-GCM per Shard

Each document shard is independently encrypted with AES-256-GCM using ephemeral keys that are never stored centrally.

At Distribution
T3

Quantum-Safe Anchoring — Dilithium-3 Lattice

The proof record anchored to the decentralized ledger uses post-quantum Dilithium-3 signatures, future-proofing against computational attacks.

At Anchoring
