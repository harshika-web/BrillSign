Evidence Chain Walkthrough

Four phases of permanent proof creation

Every BrillSign signature passes through four cryptographic phases. Each phase produces an immutable artifact — together they form an evidence package that remains legally valid decades later, independent of any platform.

4-phase cryptographic chain

End-to-End Cryptographic Trail

Each phase generates an artifact that is cryptographically linked to the next. No single component can be altered without breaking the entire chain — giving courts an unbroken, machine-verifiable chain of custody.

Phase-locked cryptographic binding Immutable ledger anchoring RFC 3161 certified timestamps
Phase 01 — Identification

Signer Identity Verified

The signer is authenticated using high-assurance methods: OIDC/SAML single-sign-on, hardware-backed MFA, or external KYC provider. A cryptographic identity assertion is generated and tied to this session only.

Phase 02 — Cryptography

Document Fingerprint Created

A SHA-256 cryptographic hash of the document's exact binary content is computed. This fingerprint is unique to this version of this document — any future alteration would produce a completely different hash.

Fingerprint locked

doc_hash: sha256:7f3a...1e09
algorithm: SHA-256
state: IMMUTABLE

Phase 03 — Binding

Signature Cryptographically Bound

The identity proof, document fingerprint, and a certified timestamp from a trusted Time Stamp Authority (TSA) are cryptographically merged into a single atomic record. No component can be changed without breaking the entire proof.

Atomic binding sealed

binding: trustchain™ v4
timestamp: RFC 3161 TSA
protocol: Dilithium-3

Phase 04 — Anchoring

Proof Permanently Anchored

The complete evidence package is anchored to BrillSign's decentralized ledger. This creates an immutable, publicly verifiable record that remains valid for decades — completely independent of the BrillSign platform's operational status.

Proof Package Anatomy

What's inside a non-repudiation proof

Proof Package

Identity Assertion

Cryptographic proof of who the signer is, without storing raw PII. Generated from authentication method and bound to this signing session only.

Document Fingerprint

SHA-256 hash of the exact binary content at signature time — any byte-level change produces a different fingerprint, making alterations instantly detectable.

RFC 3161 Timestamp

Certified timestamp from a trusted Time Stamp Authority, proving the exact moment of signing in a way that cannot be backdated or disputed.
